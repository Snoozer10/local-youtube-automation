1. Data Flow Mappings (File-to-File Relationships)

The workflow operates in a sequential chain. Below is the mapping of which
scripts (.py) read from and write to which files (.txt, .docx, .srt, .wav, .png,
and .mp4).

A. Script & Text Preparation Phase

  - automate_all.py
      - Reads:
          - youtube_urls.txt (List of target YouTube video URLs)
          - prompt.txt (Instructions for formatting the initial English
            transcript)
          - prompt_phase3.txt (Cairo Cut style guide for Gen-Z Egyptian Arabic
            translation)
      - Writes (inside the target folder youtube_runs/[Video_Title]/):
          - raw_transcript.txt (The unformatted transcript fetched from YouTube)
          - breaked_paragraphs.txt (The transcript broken into logical English
            paragraphs by Gemini)
          - [Video_Title] - Broken Paragraphs.docx (Word Document of the English
            paragraphs)
          - checkpoint.json (A temporary file to track translation progress if
            it is interrupted)
          - final_output.txt (The completed Egyptian Arabic translated script)
          - [Video_Title] - Translation.docx (Word Document of the final
            translation)

B. Voice Generation Phase

  - generate_voice.py
      - Reads:
          - voice_option_notes.txt (Acoustic and model presets like model,
            temperature, voice)
          - TTS_PROMPT.txt (Instructions telling Gemini how to plan and
            structure voice chapters)
          - youtube_runs/[Video_Title]/final_output.txt (The final translated
            Arabic script)
      - Writes (inside youtube_runs/[Video_Title]/):
          - voice_checkpoint.json (Progress tracker for voice synthesis)
          - Chapter_1.wav, Chapter_2.wav, etc. (Individual audio tracks for each
            section)
          - full_episode_voice.wav (A single stitched audio track of all
            chapters combined)

C. Audio Transcription & Alignment Phase

  - transcribe_audio.py
      - Reads:
          - voice_option_notes.txt (Determines the Whisper model size)
          - youtube_runs/[Video_Title]/Chapter_X.wav (Individual audio segments)
      - Writes (inside youtube_runs/[Video_Title]/):
          - timestamped_transcript.txt (Plain text with absolute [MM:SS]
            timestamps)
          - timestamped_transcript.srt (Subtitle format containing precise
            millisecond timestamps)

D. Image Planning & Generation Phase

  - script_image_generator.py (Batch mode) or script_image_generator_v2.py (Root
    mode)
      - Reads:
          - visual_style.txt (Aesthetic guidelines for the webcomic character)
          - visuals_plan.txt (Storyboard outline detailing scene progression)
          - youtube_runs/[Video_Title]/final_output.txt (The translated script)
          - pre_planned_prompts.txt (Cached list of generated image
            descriptions, if they exist)
      - Writes:
          - pre_planned_prompts.txt (Detailed text-to-image prompts mapped by
            index)
          - sentence_1.png, sentence_2.png, etc. (The generated background
            images)

E. Video Compilation Phase

  - compile_video.py
      - Reads:
          - youtube_runs/[Video_Title]/timestamped_transcript.srt (Time offsets
            for video frames)
          - youtube_runs/[Video_Title]/full_episode_voice.wav (The voice
            narrative)
          - generated_images/sentence_X.png (The generated images)
      - Writes:
          - youtube_runs/[Video_Title]/youtube_ready_video.mp4 (The final
            compiled video file)

2. Individual Script Analysis (What Each .py Does)

1.  automate_all.py (Core Orchestrator): Handles Phase 1 and Phase 3 of the
    script creation. It connects to a debugging Chrome browser via Playwright,
    scraps transcripts, sends them to Gemini to break them into paragraphs, and
    then iteratively translates those paragraphs into Egyptian Arabic slang
    while saving progress locally in case of network drops.
2.  generate_voice.py (Audio Producer): Navigates between Gemini Web App (to
    structure narration blocks under 150 words) and Google AI Studio's Speech
    Playground (to generate highly expressive TTS). It downloads individual
    chapters and merges them into a single master WAV file.
3.  transcribe_audio.py (Whisper Aligner): Employs OpenAI's local Whisper model
    to transcribe the generated WAV files. It outputs a .srt subtitle file
    containing the precise timings needed to display images at the correct
    moments.
4.  script_image_generator.py (Image Batch Generator): Uses Gemini's image
    generation capabilities to plan and render distinct scene images based on
    the script sentences, ensuring style consistency.
5.  compile_video.py (MoviePy Video Editor): Reads the master audio, the
    subtitle timings, and the folder of generated images, then compiles them
    into a gapless, H.264-encoded, YouTube-ready video.
6.  get_transcript.py (Stand-alone Utility): A simple, terminal-based tool that
    extracts raw English subtitles from a single YouTube URL and writes them to
    a root file (transcript.txt).
7.  paste_to_gemini.py (Utility): Directly launches Playwright to paste the raw
    text inside prompt.txt and transcript.txt into an active Gemini window.
8.  process_paragraphs.py (Alternative Translation Flow): An alternative
    translation workflow that reads from a root-level breaked_paragraphs.txt and
    writes output by pasting directly into a Google Doc (docs.new).
9.  save_and_paste_doc.py (Utility): Captures the last message generated on an
    open Gemini tab and pastes it into a blank Google Doc using system clipboard
    emulation.
10. script_pre_planner.py / script_visual_planner.py (Pre-production Planners):
    Stand-alone tools used to establish the aesthetic storyboard planning step
    and write out the planning lists before the main generation loops are run.

older versions or manual workarounds).

4. Important Integration Warning (Directory Discrepancy)

If you plan to run this pipeline end-to-end automatically, there is a path
mismatch between where images are saved and where they are read:

1.  In script_image_generator.py (Line 414): The images are saved to a folder
    inside each video's run folder: image_dir = os.path.join(subfolder,
    "generated_images") \rightarrow youtube_runs/[Video_Title]/generated_images/
2.  In compile_video.py (Line 60): The video compiler looks for images in a root
    folder: images_dir = "generated_images" \rightarrow
    [Root_Folder]/generated_images/

Recommendation to fix this: Modify line 60 of compile_video.py to search for the
images inside the specific run folder rather than the root directory:

images_dir = os.path.join(latest_run, "generated_images")

This ensures that compile_video.py successfully locates the images generated by
script_image_generator.py.
